# Step 3 — Synthesis: Generate Final Context Files from Findings

## MANDATORY EXECUTION RULES (READ FIRST):

- Read ALL findings JSON files BEFORE writing any output file
- Apply user corrections from the conversation to all generated files
- Findings from sub-agents are in `.findings/` directories — read them from disk, do not rely on memory
- Communicate progress in `communication_language`
- Delete all `.findings/` directories after successful synthesis

---

## YOUR TASK:

Read specialist findings written by sub-agents and synthesize them into final, human- and AI-readable context documents.

---

## STEP A — Load all findings

1. Re-read `{project-root}/.nakiros.workspace.yaml` to get the definitive repo list and `workspace_name`.
2. For EACH repo in `repos_to_process`: read `{repo_localPath}/_nakiros/context/.findings/arch.json` fully.
3. Read `~/.nakiros/workspaces/{workspace_name}/.findings/pm.json` fully.
4. If multi-repo: read `~/.nakiros/workspaces/{workspace_name}/.findings/inter-repo.json` if present.
5. Apply any corrections the user mentioned in the conversation before [C] was selected.

**Announce:**
```
🔍 [GENERATE-CONTEXT] Step 3 — Synthèse

Findings chargés :
{for each repo: ├─ arch.json → {repo_name}}
└─ pm.json → workspace {workspace_name}
Corrections intégrées : {oui si mentionnées, non sinon}
```

## STEP B — Generate architecture.md for each repo

**For EACH repo in repos_to_process:**

Create `{repo_localPath}/_nakiros/context/` directory if missing.

Write `{repo_localPath}/_nakiros/context/architecture.md` from the arch.json findings for that repo:

```markdown
# Architecture — {repo_name}
> Generated by Nakiros on {date}. Re-run `/nak-workflow-generate-context` to refresh.
> Analysed by Architect sub-agent.

## Entry Points
{list from findings.entry_points: file path + description}

## Tech Stack
{from findings.tech_stack: language + version, framework, key libraries, infra}

## Directory Map
{from findings.directory_map: path + role, one per line}

## Architectural Patterns
{from findings.architectural_patterns: style, state management, API style, testing, other patterns}

## Key Interfaces & Contracts
{from findings.key_interfaces: name — file — description}

## Tech Debt
{from findings.tech_debt: [SEVERITY] description — file, or "None identified"}
```

Announce after each: `✓ architecture.md → {repo_localPath}/_nakiros/context/architecture.md`

---

## STEP C — Generate pm-context.md (workspace level)

Create `~/.nakiros/workspaces/{workspace_name}/` directory if missing.

```markdown
# PM Context — {workspace_name}
```

Announce: `✓ pm-context.md → ~/.nakiros/workspaces/{workspace_name}/pm-context.md`

## STEP D — Generate inter-repo.md (if multi-repo)

If `repos_to_process` has more than 1 repo:

Write `~/.nakiros/workspaces/{workspace_name}/inter-repo.md` by cross-referencing all arch.json findings:

```markdown
# Inter-repo Relationships — {workspace_name}
> Generated by Nakiros on {date}.

## Repos in Workspace
| Name | Role | Stack | Path |
|------|------|-------|------|
{one row per repo from arch.json findings}

## Shared Packages & Contracts
{cross-reference key_interfaces across all arch.json findings — identify shared types and API boundaries}

## Event / Message Boundaries
{identify async boundaries: queues, events, webhooks between repos, or "None"}

## Dependency Graph
{which repo depends on which, derived from shared contracts and imports}
```

Announce: `✓ inter-repo.md → ~/.nakiros/workspaces/{workspace_name}/inter-repo.md`

```markdown
## Current Sprint

**Active tickets**:
{from pm findings, or "None"}

## Business Context

{from pm findings}

{from pm findings, or "None"}

```

Announce: `✓ global-context.md → ~/.nakiros/workspaces/{workspace_name}/global-context.md`
