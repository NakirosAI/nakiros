Détail de l'issue

NAK-142
Ajouter la notion d'epic et de task dans la page backlog
La page Backlog permet deja de creer des sprints et des stories, mais il manque les epics pour structurer le scope et les tasks pour decouper le travail.
Epic: Backlog UX redesign
Sprint: Sprint 14
Priority: High
Story points: 8

Sous-tâches

NAK-143 · Expose IPC create/update epic
NAK-149 · Ajouter panneau task dans le détail story
NAK-150 · Gérer les sections backlog / sprint

Critères d'acceptation

Le backlog affiche epics, stories et tasks sans confusion.
La planification sprint reste possible sans changer d'écran.
Le détail d'une story permet de créer et modifier ses tasks.

Pourquoi ce modèle marche mieux

Si ton repère mental est Jira, il faut assumer un backlog comme une liste d'issues structurée, pas comme un canvas produit. Dans ce cas, les filtres ont leur place, mais la hiérarchie des items doit être explicite et le panneau de détail doit absorber la complexité.
