EP-04 — Authentification SSO
Permettre aux organisations d'utiliser leur SSO (Okta, Azure AD, Google) pour authentifier leurs utilisateurs.
ACCEPTANCE CRITERIA
• L'admin configure un provider SAML 2.0 ou OIDC
• Les membres se connectent via SSO sans mot de passe
• Fallback email/password en cas d'échec SSO
• Si user désactivé dans l'IdP → session révoquée ≤ 5 min
• L'user est redirigé vers login avec message explicite
• Tokens locaux supprimés via safeStorage
CONTRAINTES TECHNIQUES
• Flow PKCE existant dans auth.ts
• Compatibilité Melody Auth
• Token storage via electron.safeStorage
