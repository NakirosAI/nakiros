Nakiros — Layout final : Topbar · Sidebar · Bottom bar

Topbar droite : Feedback · ⚙ — c'est tout. Le reste appartient à la bottom bar.

Responsabilité de chaque zone

Topbar — navigation & contrôles globaux

  • Workspace tabs — switcher entre projets ouverts
  • Feedback — bouton texte discret
  • ⚙ — icône seule, ouvre Global Settings (compte, agent, langue, org)
  • MCP badge → supprimé
  • Account badge (dot + prénom) → supprimé, nom visible dans Settings
  • Repo count / topology → supprimés, en bottom bar

Sidebar — navigation workspace

  • Overview · Chat · Product · Delivery · Backlog
  • Tab "Workspace" (ex-Settings) en bas — repos, git, PM, danger
  • Icône cube / package pour différencier de l'engrenage Global
  • Notification dots sur Chat si run terminé ou preview pending

Bottom bar — status ambiant système

  • Connexion cloud — online / offline / reconnecting
  • Sync workspace — synced / syncing… / error
  • Activité collègue — "Updated by Sophie" avec avatar
  • Repo count + topology — info workspace passive
  • Jamais actionnable sauf click → detail panel optionnel

Bottom bar — règles UX

  • Jamais critique — l'user peut ignorer, le flux n'est pas bloqué
  • Silencieuse par défaut — s'anime seulement quand quelque chose change
  • Clic optionnel — ouvre un detail panel (historique sync, conflits éventuels)
  • Hauteur fixe 28px — comme VS Code, ne vole pas d'espace content

Activité collègue — comportement

  • Apparaît 4–5 secondes puis disparaît silencieusement
  • Format : "[Prénom] updated [fichier court]"
  • Si plusieurs collègues en même temps → "Sophie +2 updated files"
  • Pas de toast intrusif, pas de modal — bottom bar suffit

Topbar — ce qu'on enlève définitivement

  • MCP badge — supprimé
  • 3 repos / topology — déplacé en bottom bar (faint)
  • Account badge "Connected" — supprimé, connexion dans la bottom bar
  • Résultat : topbar droite = 2 éléments (Feedback · ⚙)
